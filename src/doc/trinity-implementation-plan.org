* Purpose And Scope

- Drive the codebase from its current Golden PoC contract to the Trinity
  Engine v1 runtime contract defined in =specifications/trinity/=.
- The plan is phased so every phase produces:
  - a concrete behavioral change in the engine,
  - unit tests covering the change in isolation,
  - an integration test against the engine running as a *local process*,
  - an integration test against the engine running in a *Docker
    container* over HTTP,
  - an integration test against the engine running in a *Kubernetes*
    cluster over HTTP (via =kind=, service port-forward).
- Companion source documents:
  - =specifications/trinity/trinity.org= (canon extract)
  - =specifications/trinity/report.org= (known ambiguities)
  - =src/doc/requirement-tracking.org= (current gap analysis)
  - =src/doc/version-pins.org= (pinned versions + A-register, Phase 1+)

* Canon Ambiguities That Gate Final Acceptance

These items from =specifications/trinity/report.org= must be answered by
the canon owner before the work in later phases can be *frozen*. The
plan codes temporary implementation choices so engineering can proceed,
and flags those choices as =A1..A8= so they can be revisited.  See
=src/doc/version-pins.org= for the full dependency graph from each
=A*= item to the compiled-in constants it affects.

| ID | Topic                                      | Working assumption (revise on canon clarification)                     |
|----+--------------------------------------------+------------------------------------------------------------------------|
| A1 | IANA tzdb release                          | Pin to the =go-1.22.x embedded tzdata= release; surface in metadata.   |
| A2 | House interval ownership                   | Start-inclusive / end-exclusive on cusps, wrap between cusp 12 and 1.  |
| A3 | Design-time second selection               | Return bracket midpoint, emit =floor= to whole second in =Z= suffix.   |
| A4 | Error messages                             | Match only =error_type= in fixtures; =message= is informational prose. |
| A5 | =unsupported_input= boundary               | Reject seconds-present, sub-minute precision, multi-person payload.    |
| A6 | IANA link names                            | Accept canonical zones only; reject =US/Eastern=, =Etc/GMT+3=.         |
| A7 | Golden first-fixture oracle                | Capture once from engine, cross-check against external tool, then pin. |
| A8 | Consistency pass (Docs 05-07) not folded back | Follow =trinity.org= (mandala anchor =277.5°=, sequence starting with =38=); treat legacy =src/canon/*.json= as PoC-era. |

* Source-Tree Target Layout

Phases add packages and rename existing ones. The Golden PoC code stays
compilable during the transition, then gets removed at the end. Proposed
end-state under =src/mademanifest-engine/pkg=:

- =canon/=           canonical constants + version pins (promoted)
- =trinity/input/=   Trinity input DTO + validator + rejection classifier
- =trinity/output/=  Trinity success + error DTOs + serializer
- =astro/=           astrology calculation producing Trinity objects
- =hd/calc/=         HD personality + design snapshot calculators
- =hd/structure/=    channels, centers, definition, type, authority,
                     profile, incarnation_cross
- =genekeys/=        Trinity Gene Keys derivation
- =engine/=          orchestrator returning Trinity envelopes
- =httpservice/=     HTTP adapter emitting Trinity envelopes
- =golden/=          Trinity golden pack harness
- =emit_golden/=     *removed* at Phase 12

* Cross-Cutting Test Harness (built in Phase 0, used in every phase)

** Unit tests
- Go =testing= standard. Table-driven tests. No network. No filesystem
  outside the tested fixture tree.

** Local-process integration tests
- New target =make test-integration-local=.
- Launches =cmd/httpserver= on a random loopback port with
  =CANON_DIRECTORY= and =SE_EPHE_PATH= pointing at the repo checkout.
- Test binary lives under
  =src/mademanifest-engine/integration/local_test.go= and calls
  =GET /healthz= and =POST /manifest= with real =net/http=.

** Docker integration tests
- New target =make test-integration-docker=.
- Builds the production =src/Dockerfile= image as
  =mademanifest-engine:itest=.
- Runs =docker run -d -p 18080:8080= of that image; waits on
  =GET /healthz=.
- Test binary lives under =src/mademanifest-engine/integration/docker_test.go=
  (guarded with build tag =integration_docker=) and hits the container
  over HTTP with the same golden pack as the local-process tests.

** Kubernetes integration tests
- New target =make test-integration-k8s=.
- Spins up a =kind= cluster =kind create cluster --name trinity-itest=.
- =kind load docker-image mademanifest-engine:itest=.
- =kubectl apply -k src/deploy/kubernetes/= with the image pin
  overridden for the test tag; waits on deployment readiness.
- Opens =kubectl port-forward svc/mademanifest 18081:80= as a sidecar
  process; test binary lives under
  =src/mademanifest-engine/integration/k8s_test.go= (build tag
  =integration_k8s=) and drives the service HTTP API exactly like the
  docker test.
- Test teardown deletes the =kind= cluster.

** Shared fixture set
- Every phase extends =src/golden/trinity/= with fixtures labelled
  =valid_baseline=, =valid_edge=, =invalid_input=, =incomplete_input=,
  =unsupported_input=, =regression_sentinel= per Document 10. The same
  fixture set drives local, docker, and k8s integration suites.

* Phase 2 – Trinity Input DTO And Validator

** Deliverables
- =pkg/trinity/input/input.go=:
  =type Payload struct { BirthDate, BirthTime, Timezone string; Latitude, Longitude float64 }=.
- =pkg/trinity/input/validator.go= implements Document 04:
  - JSON decode with =DisallowUnknownFields=.
  - Explicit rejection of numeric-looking strings (custom unmarshal).
  - IANA canonical-zone check (list compiled from =tzdata=; rejects
    link names per A6).
  - Gregorian date + 24-hour minute format regexes.
  - Classifier returns =RejectIncomplete=, =RejectInvalid=, or
    =RejectUnsupported= (A5) carrying an enum field name.
- =engine.Run= replaces =process_input.ProcessInput= as the input entry
  point; the Golden PoC path is left intact but is no longer exercised
  by the HTTP handler.

** Unit tests
- Table-driven =validator_test.go= case per rule:
  - missing each field → =incomplete_input=
  - numeric field as string → =invalid_input=
  - malformed date (=1990-02-30=, =90-04-09=) → =invalid_input=
  - seconds present (=18:04:00=), or sub-minute precision →
    =unsupported_input= (A5)
  - IANA abbreviation (=CET=) → =invalid_input=
  - IANA link name (=US/Eastern=) → =invalid_input= (A6)
  - lat > 90, lon out of range → =invalid_input=

** Integration
- =POST /manifest= with every rejection category: envelope has
  =status="error"=, correct =error_type=, non-empty =message=, and
  complete =metadata=. Tests assert only the =error_type= (A4) and
  metadata block.
- Runs under local, docker, k8s harnesses.

* Phase 3 – Trinity Output Envelope And Error Path

** Deliverables
- =pkg/trinity/output/envelope.go= types:
  - =SuccessEnvelope= (=status, metadata, input_echo, astrology, human_design, gene_keys=).
  - =ErrorEnvelope= (=status, metadata, error{error_type,message}=).
  - =Metadata= with the five pinned version fields and *nothing else*.
- Stable JSON serializer with pinned key order. Implementation option:
  struct tags + ordered map encoder. Round trip tested.
- HTTP handler replaced:
  - Status code policy: =200= for =success=; =400= for
    =incomplete_input= + =invalid_input=; =422= for =unsupported_input=;
    =500= for =canon_conflict= + =execution_failure=.
- Longitude serialized as =float64= rounded to 6 decimals;
  =design_time_utc= formatted =RFC 3339 UTC= with whole-second precision
  (A3 rounding rule = floor).

** Unit tests
- =envelope_test.go=: marshal / unmarshal preserves order; missing
  metadata fields rejected; numeric fields kept numeric.
- Handler tests via =httptest.NewRecorder= for every =error_type=.

** Integration
- Local, docker, k8s suites hit:
  - a valid baseline → =200= with full envelope keys present in order.
  - each error type → correct status code + envelope.

* Phase 5 – Human Design Design-Time Solver (canonical form)

** Deliverables
- =pkg/hd/calc/designtime.go= replaces the parameter-driven solver:
  - constants hard-coded from canon (=88.0°=, stop thresholds).
  - backward-only search; initial bracket centered ~88 days back.
  - pure bisection (no secant fallback).
  - returns UTC time; emission applies A3 rounding in Phase 3 output
    layer.
- Remove =DesignTimeSolver= struct from the engine API surface.

** Unit tests
- Known-seed test using a hand-crafted =sunFunc= (linear over 365 days)
  produces the mathematically exact target within tolerance.
- Monotonicity test: bracket shrinks strictly; final width ≤ 1 second.
- Forbidden shortcut test: ensure no path subtracts a fixed 88 days
  (inspection of the sun-func call sequence; we capture the minimum
  number of iterations and require ≥ 20 calls, defeating the trivial
  shortcut).

** Integration
- Regression sentinel fixture derived from baseline: assert
  =human_design.system.design_time_utc= equals the canonical value to
  the whole second.

* Phase 9 – Determinism Cleanup

** Deliverables
- Remove all runtime-affecting environment variables (=SE_NODE_POLICY=,
  any fallback path tricks). =SE_EPHE_PATH= remains a *deployment*
  setting but is validated on boot and its resolved absolute path is
  surfaced in =/version= for diagnosis (never in =metadata=).
- =pkg/canon= becomes the single source of every calculation
  constant; engine refuses to boot if the constants fail their
  self-checks.
- Gate-sequence file becomes a sanity check against the compiled-in
  =GateOrder= rather than a configuration source.

** Unit tests
- Boot-time self-check fails fast on tampered canon files.

  confirms the engine refuses to start.

** Integration
- Docker image built with =--env SE_NODE_POLICY=true= still produces
  canonical output. Same for k8s pod with the same environment.

* Risks And Mitigations

- *Canon ambiguities freeze mid-flight*: A1..A7 may force rework of
  Phases 2, 3, 4, 5. Each affected commit is tagged with the =Ax= id
  so revision is surgical.
- *Oracle self-freeze* (A7): we explicitly cross-check baseline
  fixtures with an external reference before pinning — no self-pinning
  allowed.
- *Environmental drift*: kind image and Docker base images pinned by
  digest; =go.mod=, =go.sum=, and =swisseph= tarball already committed.
- *K8s suite flakiness in CI*: the kind cluster startup is long; only
  runs on merge to =main= and nightly, not on every PR.
