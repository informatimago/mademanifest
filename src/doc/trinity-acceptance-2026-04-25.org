* Canon Ambiguity Status (A1..A8)

All eight register entries from
=specifications/trinity/report.org= remain =UNRESOLVED= at this
release.  The engine ships working assumptions documented in
=src/doc/version-pins.org= and tagged in code with the matching
=A*= identifier.  The canon owner must address each item before the
next release bumps =canon_version= past =trinity-v1-rev-0=.

| ID | Topic                                       | Status     | Working assumption                                    |
|----+---------------------------------------------+------------+-------------------------------------------------------|
| A1 | IANA tzdb release                          | UNRESOLVED | Pin to Go 1.22 embedded =time/tzdata= (release 2023c) |
| A2 | House interval ownership                   | UNRESOLVED | Start-inclusive / end-exclusive on cusps; 12→1 wrap   |
| A3 | Design-time second selection               | UNRESOLVED | Bracket midpoint, floor to whole second on emit       |
| A4 | Error message catalog                      | UNRESOLVED | Match =error_type= only; =message= informational      |
| A5 | =unsupported_input= boundary               | UNRESOLVED | Reject seconds-present, sub-minute, multi-person      |
| A6 | IANA link names                            | UNRESOLVED | Reject =US/Eastern=, =Etc/GMT+3= and other aliases    |
| A7 | Golden first-fixture oracle                | UNRESOLVED | Capture once + cross-check published reference         |
| A8 | Doc 05/06/07 consistency pass               | UNRESOLVED | Follow =trinity.org= (anchor 277.5°, gate 38 first); legacy JSON canon already retired in Phase 12 |

For each entry, the canon owner's review is gated on:
- A1, A6 — IANA tzdb release confirmation (pin to a specific year-letter).
- A2 — explicit canon language for boundary ownership on cusps.
- A3 — choice between lower bound, midpoint, upper bound, or
       nearest-second emission of =design_time_utc=.
- A4 — error-message text catalog or explicit confirmation that
       only =error_type= is contractual.
- A5 — explicit list of scenarios that must produce
       =unsupported_input= rather than =invalid_input=.
- A7 — review and freeze of the eleven success-case oracles in
       =src/golden/trinity/=.
- A8 — confirmation that =trinity.org= mandala anchor 277.5° is
       the canonical value (Phase 12 retirement assumed yes).

* Acceptance Decision

Per =trinity.org= §"Delivery And Acceptance" the minimum
acceptance conditions are:

- [X] all required in-scope features implemented (Phases 0..13).
- [-] no canon-critical ambiguity remains unresolved in the
  delivered scope — *deferred*: A1..A8 carry =UNRESOLVED= status;
  the engine ships working assumptions documented in =version-pins.org=.
- [X] required metadata present and correct
  (=output.CurrentMetadata= asserted by every test pass).
- [X] required golden tests pass on local, docker, and kind.
- [X] output matches canon exactly for the success cases the
  oracle covers (per A7 the canon owner has not yet approved the
  freeze; the per-section oracles cross-check against the
  published Schiedam PoC chart and against trinity.org canonical
  decision trees).
- [X] no undisclosed canon deviation — every working assumption is
  tagged with its =A*= identifier in code and =version-pins.org=.
- [X] no out-of-scope shortcuts simulating compliance.

Conditional acceptance: =v1.0.0-trinity= ships pending the canon
owner's resolution of A1..A8.  Resolution of any =A*= item that
changes a compiled-in constant (anchor, error_type mapping,
unsupported_input boundary, etc.) requires a new =canon_version=
bump per Document 08; the Phase 11 golden pack must be regenerated
in lockstep.
