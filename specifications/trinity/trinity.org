** Validation Rules
- Validate before any calculation:
  - presence of all required fields
  - exact field format compliance
  - type compliance
  - numeric range compliance
  - IANA-compliant timezone format
  - Gregorian date validity
  - 24-hour minute time validity
- Strict numeric typing:
  - numeric fields must be numeric values, not strings
  - no silent coercion of numeric-looking strings
- No optional canonical input fields in v1.
- Extra upstream fields are non-authoritative and must not affect results.

** Normalization Assumptions
- Before engine submission:
  - location already resolved to unique latitude and longitude
  - timezone already resolved to a valid IANA identifier
  - date already resolved to Gregorian YYYY-MM-DD
  - time already normalized to HH:MM
  - ambiguity, alias handling, and UI cleaning already completed upstream
  - payload already reduced to one birth event

** Rejection Rules
- =incomplete_input=
  - missing one or more required fields
- =invalid_input=
  - malformed date or time
  - invalid timezone string
  - out-of-range latitude or longitude
  - numeric field supplied as string
- =unsupported_input=
  - structurally valid input outside supported v1 scope

** No Silent Repair
- The engine must not silently:
  - infer missing fields
  - correct malformed values
  - coerce ambiguous inputs
  - map timezone aliases
  - resolve place names
  - truncate unexpected time formats
  - substitute defaults

* Calculation Canon
** Shared Pre-Calculation Steps
- Validate input.
- Construct local birth datetime from date, time, timezone.
- Convert to UTC using the pinned timezone standard.
- Derive Julian Day or equivalent astronomical time input.
- Use provided latitude and longitude as authoritative coordinates.

** Angular Mechanics
- Canonical longitude range: =[0.0, 360.0)=.
- Normalize by modular arithmetic.
- =360.0= normalizes to =0.0=.
- Opposition derivation uses canonical 180-degree opposition logic.
- Cyclic interval handling must be deterministic.

** Astrology
- Query or derive longitudes at the birth moment.
- Earth longitude = Sun longitude + 180 degrees modulo 360.
- Sign placement is derived from the tropical zodiac intervals.
- House calculation uses:
  - birth moment
  - latitude
  - longitude
  - Placidus
- Return:
  - ascendant
  - midheaven
  - house cusps 1-12
  - supported astrology objects in canonical order

** Human Design Design-Time Derivation
- Design time is a second astronomical moment before birth.
- Design-time target rule:
  - find the prior moment where Sun longitude is exactly 88.0 degrees earlier than at birth
  - target longitude = normalize(birth_sun_longitude - 88.0)
- Search direction:
  - backward in time only
- Initial bracket:
  - centered around approximately 88 days before birth
- Root-finding method:
  - iterative bisection
- Stop conditions:
  - absolute Sun-longitude difference from target < 0.0001 degrees
  - or remaining interval width < 1 second
- Forbidden shortcuts:
  - fixed 88-day subtraction
  - fixed 3-month subtraction
  - rough platform imitation
  - average ephemeris approximation
  - precomputed lookup shortcut

** Human Design Gate And Line Mapping
- Mandala parameters:
  - anchor defined in Document 06
  - gate width = 5.625 degrees
  - line width = 0.9375 degrees
  - direction = ascending ecliptic longitude
  - intervals are start-inclusive and end-exclusive
  - gate order is a fixed lookup list
- Relative mandala position:
  - normalize longitude
  - subtract canonical anchor
  - normalize result into =[0.0, 360.0)=
- Gate assignment:
  - floor(relative_position / 5.625)
  - use result as zero-based index into canonical gate order
- Line assignment:
  - compute offset inside gate segment
  - floor(offset / 0.9375)
  - convert to line number 1..6
- Boundary rule:
  - an interval end belongs to the next segment
- Gene Keys uses exactly the same gate and line results as Human Design.
